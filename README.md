# Xls Data View

基于 TypeScript 的本地表格导入与筛选工具。

## 功能

- 三页面结构：
  - 结果观察：查看当前数据库数据 + 多筛选器
  - 数据导入：拖拽导入，自动行主键入库
  - 设置：清空全部数据
- 支持导入 `.xls / .xlsx / .csv`
- 同系列导入时，校验表头一致
- 不要求业务主键：每行自动分配唯一行 ID，完整保留所有行（包括重复值行）
- 同系列再次导入时按新文件整表覆盖，避免键冲突导致丢行
- 多筛选器组合查询（包含、等于、前缀、大于、小于）
- 大文件导入进度：解析阶段与入库阶段均显示进度条，避免页面“未响应”感知
- 支持“多数据组”比对分析：
  - 默认不自动比对：手动配置多个数据组后点击“开始比对”
  - 每个数据组可指定日期、陷阱、地图（地图可选）
  - 参数支持多选（如使用次数、出售次数、总伤害），未选择则比对所有数值参数
  - 输出柱状图 / 折线图 + 结果明细表

## 技术栈

- React + TypeScript + Vite
- `xlsx` 解析 xls/xlsx/csv
- Dexie（IndexedDB）本地存储

## 启动

```bash
npm install
npm run dev
```

## Windows 打包（桌面应用）

已接入 Electron + electron-builder，可打包独立 Windows 应用。

### 首次准备

```bash
npm install
```

### 本地桌面调试

```bash
npm run dev:desktop
```

### 生成 Windows 安装包（NSIS + Portable）

```bash
npm run pack:win
```

输出目录：`release/`

### 仅生成便携版

```bash
npm run pack:win:portable
```

说明：
- 建议在 Windows 系统上执行 `pack:win`，产物兼容性最好。
- 应用数据默认保存在 Electron 用户目录（长期使用可持续保留本地数据库）。

## 使用说明

1. 进入“数据导入”页，输入系列名并拖拽文件
2. 点击“导入并覆盖同系列”
3. 进入“结果观察”页查看数据并添加筛选器
4. 在“分组比对”中配置多个数据组并点击“开始比对”
5. 查看图表和结果表

## 导入策略说明

- 自动行主键：每行都保留，不按业务列去重
- 同系列覆盖：再次导入同系列时，用新文件完整替换旧数据
- 同系列增量追加：仅追加数据库中不存在的整行，重复整行自动跳过
- 如果你需要保留历史版本，请使用不同系列名（例如 `trap_daily_2025_09_03`）

## 清空全部数据

点击“清空全部数据”会删除 IndexedDB 中所有系列与行数据，且不可恢复。
